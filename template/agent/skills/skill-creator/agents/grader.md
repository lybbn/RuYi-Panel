# 技能评估指导

评估一个技能的质量，从触发准确性和执行效果两个维度进行。

## 评估维度

### 1. 触发准确性

评估 description 是否能让 AI 在正确的场景下触发技能。

**检查项：**
- 应触发的场景是否被 description 覆盖
- 不应触发的场景是否被 description 排除
- description 是否过于宽泛导致误触发
- description 是否过于狭窄导致漏触发

**测试方法：**
列举3-5个应该触发此技能的用户输入，和2-3个不应触发的用户输入，检查 AI 是否能正确判断。

### 2. 指令完整性

评估 SKILL.md 正文是否提供了足够的操作指导。

**检查项：**
- 操作步骤是否完整，有无遗漏
- 工具调用是否正确（工具名、参数）
- 输出格式是否清晰
- 边界情况是否有处理指导

### 3. 平台适配性

评估技能在 Linux 和 Windows 上的兼容性。

**检查项：**
- 是否明确标注了平台差异
- Linux/Windows 命令是否正确
- 路径格式是否适配
- 是否优先使用了面板工具（已处理平台差异）

### 4. 安全性

评估技能是否存在安全风险。

**检查项：**
- 危险操作是否有确认步骤
- 是否可能泄露敏感信息
- 是否包含可能危害系统的操作
- 脚本是否有路径遍历等安全漏洞

## 评估报告格式

```markdown
# 技能评估报告

## 基本信息
| 项目 | 值 |
|------|------|
| 技能名称 | xxx |
| 评估日期 | xxx |
| 评估结果 | 通过/需改进/不通过 |

## 触发准确性
- 应触发场景覆盖：x/x
- 不应触发排除：x/x
- 问题：[具体问题]

## 指令完整性
- 步骤完整性：完整/部分缺失
- 工具调用准确性：准确/有误
- 输出格式：清晰/需改进
- 问题：[具体问题]

## 平台适配性
- Linux 支持：完整/部分/不支持
- Windows 支持：完整/部分/不支持
- 问题：[具体问题]

## 安全性
- 危险操作确认：有/无
- 敏感信息保护：有/无
- 问题：[具体问题]

## 改进建议
1. [按优先级排列的具体建议]
```

## 评分标准

| 等级 | 分数 | 说明 |
|------|------|------|
| 优秀 | 90-100 | 触发准确、指令完整、平台适配、安全可靠 |
| 良好 | 70-89 | 基本功能完整，有少量改进空间 |
| 需改进 | 50-69 | 存在明显问题，需要修改后才能使用 |
| 不通过 | 0-49 | 存在严重问题，需要重写 |
