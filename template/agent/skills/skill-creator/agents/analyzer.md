# 技能分析指导

分析技能的执行效果，发现模式、问题和改进机会。

## 分析流程

### 第一步：收集执行数据

1. 查看技能的调用历史和成功率
2. 收集用户反馈和评价
3. 检查工具调用日志中的错误记录

### 第二步：触发模式分析

1. **漏触发** — 应该使用技能但未触发的场景
   - 原因通常是 description 未覆盖该场景
   - 解决：在 description 中补充触发关键词

2. **误触发** — 不应使用技能但被触发的场景
   - 原因通常是 description 过于宽泛
   - 解决：在 description 中增加排除条件或缩小范围

3. **边界模糊** — 既可以触发也可以不触发的场景
   - 需要明确技能的边界定义
   - 解决：在 description 中明确适用范围

### 第三步：执行效果分析

1. **步骤遗漏** — AI 跳过了某些步骤
   - 原因：步骤描述不够明确，或步骤间逻辑不清
   - 解决：加强步骤间的逻辑关联，解释每步的必要性

2. **工具调用错误** — AI 调用了错误的工具或参数
   - 原因：工具名/参数描述不准确
   - 解决：确保工具名与注册名完全一致，参数说明清晰

3. **输出格式偏差** — 输出不符合预期格式
   - 原因：格式模板不够明确或约束力不足
   - 解决：提供更详细的格式模板，使用示例说明

4. **重复工作** — 多次执行都产生相同的辅助操作
   - 原因：缺少预置的脚本或模板
   - 解决：将重复操作提取到 scripts/ 目录

### 第四步：跨技能分析

1. **功能重叠** — 多个技能覆盖相似场景
   - 解决：明确各技能的边界，或合并为一个更通用的技能

2. **功能缺口** — 常见场景没有对应技能
   - 解决：创建新技能填补缺口

3. **依赖关系** — 技能间存在隐含依赖
   - 解决：在技能说明中明确依赖关系

```markdown
|------|------|

| 调用次数 | xx |
| 成功率 | xx% |

## 触发分析
| 类型 | 数量 | 占比 | 典型案例 |
|------|------|------|----------|
| 正确触发 | xx | xx% | ... |
| 漏触发 | xx | xx% | ... |
| 误触发 | xx | xx% | ... |

## 执行分析
| 问题类型 | 出现次数 | 严重程度 | 典型案例 |
|----------|----------|----------|----------|
| 步骤遗漏 | xx | 高/中/低 | ... |
| 工具错误 | xx | 高/中/低 | ... |
| 格式偏差 | xx | 高/中/低 | ... |

## 改进建议
1. [按影响优先级排列]
```

## 分析原则

- **基于数据** — 结论必须有数据支撑，不凭感觉
- **关注模式** — 重复出现的问题比偶发问题更重要
- **考虑泛化** — 改进应适用于广泛场景，而非仅针对特定案例
- **量化评估** — 尽量用数字描述问题严重程度
